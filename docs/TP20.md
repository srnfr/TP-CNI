## But du TP

Publier une application interne en HTTP puis HTTPS avec **Gateway API**.

Contrairement au TP15 (Ingress NGINX), ici nous utilisons directement la Gateway API avec l'implémentation Cilium.
Sur DigitalOcean, la `GatewayClass` Cilium est déjà installée.

## Etape 0 : Initialisation du numero de groupe

Positionner une variable qui contient votre numéro de groupe :

```shell
GRP=X
echo $GRP
X
```

<ins>**Bien sûr utiliser la bonne valeur de X**</ins>

## Etape 1 : Vérifier la GatewayClass Cilium

```shell
kubectl get gatewayclass
```

Vous devez voir une classe `cilium` (ou équivalent Cilium) en état accepté.

## Etape 2 : Déployer une application de test

```shell
kubectl create deployment demo --image=nginx --port=80
kubectl expose deployment demo --name demo-service --port 80 --target-port 80
kubectl get pods
kubectl get svc demo-service
```

## Etape 3 : Publication HTTP avec Gateway API

Déployons une `Gateway` et une `HTTPRoute`.

```shell
cat > gateway-http.yaml << 'EOF'
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: demo-gateway
spec:
  gatewayClassName: cilium
  listeners:
    - name: http
      protocol: HTTP
      port: 80
      hostname: groupeX.randco.eu
---
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: demo-route
spec:
  parentRefs:
    - name: demo-gateway
  hostnames:
    - groupeX.randco.eu
  rules:
    - matches:
        - path:
            type: PathPrefix
            value: /
      backendRefs:
        - name: demo-service
          port: 80
EOF
```

```yaml
# gateway-http.yaml
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: demo-gateway
spec:
  gatewayClassName: cilium
  listeners:
    - name: http
      protocol: HTTP
      port: 80
      hostname: groupeX.randco.eu
---
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: demo-route
spec:
  parentRefs:
    - name: demo-gateway
  hostnames:
    - groupeX.randco.eu
  rules:
    - matches:
        - path:
            type: PathPrefix
            value: /
      backendRefs:
        - name: demo-service
          port: 80
```

Appliquer en remplaçant `X` par votre groupe :

```shell
sed "s/X/$GRP/g" gateway-http.yaml | kubectl apply -f -
```

Vérifier le statut :

```shell
kubectl get gateway demo-gateway
kubectl get httproute demo-route -o wide
```

Récupérer l'adresse exposée et tester :

```shell
kubectl get gateway demo-gateway -o jsonpath='{.status.addresses[0].value}'
curl -H "Host: groupe${GRP}.randco.eu" http://<GATEWAY_IP>/
```

Si la résolution DNS est en place (tâche formateur), vous pouvez aussi tester :

```shell
curl http://groupe${GRP}.randco.eu
```

## Etape 4 : Publication HTTPS avec Gateway API

Dans ce TP, on active HTTPS avec un certificat auto-signé (simple et autonome).

Créer un certificat TLS local :

```shell
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout tls.key \
  -out tls.crt \
  -subj "/CN=groupe${GRP}.randco.eu/O=TP-CNI"
```

Créer le secret TLS Kubernetes :

```shell
kubectl create secret tls demo-gateway-tls --cert=tls.crt --key=tls.key
```

Mettre à jour la `Gateway` pour écouter aussi en HTTPS :

```shell
cat > gateway-https.yaml << 'EOF'
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: demo-gateway
spec:
  gatewayClassName: cilium
  listeners:
    - name: http
      protocol: HTTP
      port: 80
      hostname: groupeX.randco.eu
    - name: https
      protocol: HTTPS
      port: 443
      hostname: groupeX.randco.eu
      tls:
        mode: Terminate
        certificateRefs:
          - kind: Secret
            name: demo-gateway-tls
EOF
```

```yaml
# gateway-https.yaml
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: demo-gateway
spec:
  gatewayClassName: cilium
  listeners:
    - name: http
      protocol: HTTP
      port: 80
      hostname: groupeX.randco.eu
    - name: https
      protocol: HTTPS
      port: 443
      hostname: groupeX.randco.eu
      tls:
        mode: Terminate
        certificateRefs:
          - kind: Secret
            name: demo-gateway-tls
```

Appliquer en remplaçant `X` :

```shell
sed "s/X/$GRP/g" gateway-https.yaml | kubectl apply -f -
```

Tester HTTPS :

```shell
curl -k -H "Host: groupe${GRP}.randco.eu" https://<GATEWAY_IP>/
```

`-k` est nécessaire car le certificat est auto-signé.

## Cleanup

```shell
kubectl delete httproute demo-route
kubectl delete gateway demo-gateway
kubectl delete service demo-service
kubectl delete deployment demo
kubectl delete secret demo-gateway-tls
rm -f tls.crt tls.key
```
